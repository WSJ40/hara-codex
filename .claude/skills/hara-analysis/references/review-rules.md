# HARA 评审规则

## Blocking 规则

以下问题必须修复后才能交付：

- 字段缺失、sheet 名错误、枚举不在允许值中。
- `derive_mf.子功能` 偏离源文档功能清单，或把功能内部逻辑误拆成独立功能。
- sheet1 故障描述不是功能行为偏离，而是根因、场景、危害事件或安全机制。
- MF、Milf、HARA、SG 编号重复或不符合规则。
- sheet2 故障描述不能追溯到 sheet1。
- sheet2 把当前故障偷换成另一个引导词或另一个故障，例如把 `EPB夹紧力过大` 当成 `EPB非预期拉起`。
- HARA 故障/整车危害不能追溯到 sheet2。
- ASIL 与 S/E/C 矩阵不一致。
- 非 QM HARA 行缺少安全目标、安全状态或 FTTI。
- QM HARA 行填写了安全目标、安全状态或 FTTI。
- 危险事件中故障不是必要原因。
- 车辆状态与车速明显矛盾，例如驻车 + 高速。
- 场景超出功能操作域且没有合理越域故障解释。
- 风险对象不在车辆危险路径内。
- 道路类型与风险对象/特殊要素互相冲突，例如高速公路默认出现行人横穿、骑行者或城市路口。
- 非 QM 条目没有进入 `sg_sum`。
- 非 `无危害` 故障的有效 HARA 场景少于 3 条且没有明确排除理由。

## Warning 规则

以下问题应尽量修复，若保留需要在评审报告说明：

- 使用 `ALL` 或 `不涉及` 过多，导致场景不可评审。
- 只通过改变天气、道路名或车速生成重复场景。
- 同一故障场景没有覆盖常见高暴露、最高合理严重度、低可控性/边界工况。
- sheet1 漏掉明显的作用量过大/过小，例如亮度、夹紧力、制动力、扭矩、角度、速度或持续时间。
- sheet1 把驾驶员主动离散请求误写成过早，而不是非预期激活。
- sheet1 的功能丧失按多个前置条件重复列出冗余故障，而不是概括为一个功能级偏离。
- sheet1 的卡滞没有写明卡滞状态。
- sheet1 的方向错误没有写明请求方向和实际执行方向相反。
- `E-解释` 没有说明频率或使用占比。
- `可能的后果('S'的理由)` 没有说明伤害对象和伤害机制。
- `C-解释` 没有说明感知、反应时间、空间或驾驶员是否在车上。
- 安全目标包含实现词：传感器、算法、报文、CAN、驱动、诊断、DTC、软件、硬件、冗余。
- FTTI 只有数字但没有基于场景或危害的解释来源。

## 反模式

### AP-001 天气替代因果

雨、雪、雾、夜间、低附着不能单独造成 item 故障导致的危害。它们只能作为暴露或严重度/可控性的影响因素。

### AP-002 驻车高车速

车辆状态为驻车、停车熄火、停车未熄火时，车速通常应为静止状态或低速蠕行。若写高速，必须是非预期移动或功能越域并有解释。

反向也成立：车辆状态为前向匀速、前向加速、转弯、变道、超车等行驶状态时，车速不能写静止状态。

### AP-003 风险对象错位

危险事件中的受伤人员必须位于危险路径内。前方障碍物不能支持“对向车辆乘员受伤”，后方来车不能支持“行人被前照灯照明不足撞击”。

### AP-004 普通交通风险冒充功能安全风险

如果没有 item 故障，同样场景仍会发生同样危险事件，则不是有效 HARA 危险事件。

### AP-005 安全目标设计化

安全目标不能写成具体实现机制。设计机制应出现在功能安全概念或技术安全需求中，不应作为 SG。

### AP-006 过大故障偷换成非预期激活

作用量过大只描述请求动作发生时输出过高，不能为了构造危害加入“非预期动作”。如果当前故障只会造成零件磨损、寿命下降、能耗或舒适性问题，应映射为 `无危害`。

### AP-007 近光灯按远光灯夸大

近光灯异常不能默认等同于远光灯强眩目。视野不足、误判或短暂延迟类场景要检查驾驶员是否通常能提前发现、是否有心理预期和可用应对措施；常见起步/傍晚场景往往是 QM。

### AP-008 EPB驻车溜车直接按最高E/C

EPB拉起失败、夹紧力不足、响应过晚、请求拉起却释放等驻车溜车场景，必须有坡度、外力、驱动蠕行或已有保持力释放作为纵向力来源。驾驶员仍在车上时通常可通过脚刹、P挡或重新操作补救，不应轻易给 C3；暴露率应针对坡道/行人/外力组合场景评估，不能把“每次停车”直接等同于该危险场景的 E4。

### AP-009 道路类型与风险对象冲突

高速公路、城市快速路等封闭或快速通行道路，不应默认存在行人横穿、骑行者、红绿灯路口或环岛路口。若确实需要弱势交通参与者，必须说明施工区域、事故场景、服务区、收费站、应急车道、路肩、救援或养护等特殊前提。否则应改用更符合道路环境的危险对象，例如前方停驶车辆、慢行车辆、遗落货物、障碍物、护栏或前车乘员。

## 最终评审提问

逐行检查：

1. 故障描述是否来自正常功能偏离？
2. 整车危害是否是车辆级状态？
3. 车辆状态、车速、驾驶员是否在车上是否运动学一致？
4. 道路类型、道路条件、特殊要素、风险对象是否属于同一种道路拓扑？
5. 环境条件是否支撑危害事件中的夜间、日间、雨雪、湿滑、低能见度等描述？
6. 风险对象是否真的位于车辆危险路径内？
7. 危害事件是否和当前故障、当前整车危害属于同一风险机理？
8. 移除故障后，危险事件是否消失或显著降低？
9. S/E/C 理由是否只使用当前场景事实？
10. ASIL 是否由工具计算？
11. SG 是否覆盖最高 ASIL 事件？
