# S/E/C 和 ASIL 指南

## 输出方式和判断模型

SEC 评级是对当前危险事件的首次工程判断，不是后处理修订。输出理由中禁止出现“评级校正”“修正”“按规则压低/提高”“根据指南调整”等措辞。

每一项理由都必须从本行场景事实推出，不要照抄规则文本：

- S：说明风险对象、碰撞/伤害机制、速度或能量依据；不要用 E 或 C 反推 S。
- E：说明这类操作场景在合理预期使用中的暴露频率；不要把故障发生概率算进 E。
- C：说明驾驶员或相关人员如何感知异常、可用反应时间、可用操作、道路空间、驾驶员是否在控制回路、其他交通参与者是否能避让。

SEC 理由不得引入本行场景字段和附加条件中没有的时序、操作状态或环境条件。若 E 解释使用“起步”“傍晚开灯”“进入暗环境”等前提，场景的 `车辆状态`、`环境条件`、`特殊要素` 或 `附加条件` 必须已经体现这些前提；否则必须围绕已有场景给出暴露理由，例如“夜间无路灯城市道路行驶且驾驶员因刚接手车辆、误以为已开灯等原因未及时发现近光灯未点亮”。

如果正确评级依赖某个更具体的场景前提，应优先在场景生成/评审阶段把该前提写入场景，而不是在 SEC 解释阶段临时补充。

对于同一故障，不同场景可以有不同评级；不要为了保持同一故障评级一致而忽略场景差异。对于低风险场景，要明确说明为什么是低暴露、低严重度或较易控制，而不是删除低风险场景。

## Severity S

S0：无人员伤害。仅舒适性、可用性、财产损失或信息体验问题。

S1：轻度和中度伤害。通常为低速、低能量事件，可能导致擦伤、挫伤、轻微脑震荡、轻度骨折等，一般可恢复。

S2：严重和危及生命的伤害，但通常仍有生存可能。可能包含严重骨折、内脏损伤、严重头部创伤、多根肋骨骨折、脊柱损伤等。

S3：危及生命和致命伤害。高速严重碰撞、翻滚、火灾、电击、车辆失控撞击弱势交通参与者等可评为 S3。

速度参考只作为锚点，不要机械套用：

- 车车正碰/追尾：小于 4 km/h 常为 S0；4-20 km/h 常为 S1；20-40 km/h 常为 S2；大于等于 40 km/h 可为 S3。
- 车车侧碰：小于 2 km/h 常为 S0；2-8 km/h 常为 S1；8-16 km/h 常为 S2；大于等于 16 km/h 可为 S3。
- 与行人、自行车、电动车、摩托车等弱势交通参与者碰撞：0 为 S0；0-8 km/h 常为 S1；8-15 km/h 常为 S2；大于等于 15 km/h 可为 S3。

## Exposure E

E0：不可置信。场景不属于合理预期使用。

E1：极低概率。通常小于 0.4 小时/年，或绝大多数驾驶员一年不到一次。

E2：低概率。0.4-4 小时/年，或一年几次。例如平行泊车、坡道起步、少量雨雪、学校区域。

E3：中等概率。4-40 小时/年，或月度出现。例如倒车、停车场、住宅区、乡村道路、郊区、弯道、夜间驾驶、混合交通。

E4：高概率。大于 40 小时/年，或几乎每次驾驶。例如直线道路、高速/城市通勤、正常天气、白天、高速公路巡航。

E 取决于操作场景暴露，不取决于故障概率。

## Controllability C

C0：一般可控。伤害可通过常规行为避免，或不会导致不可接受人身伤害风险。

C1：简单可控。大多数驾驶员通过直接常规反应可以避免伤害；提示明显，反应时间通常大于 5 秒。

C2：通常可控。部分驾驶员可通过及时适当反应避免伤害；反应时间通常 1-5 秒，需要较高注意力、空间或操作能力。

C3：难以控制或不可控。多数驾驶员或相关人员难以及时避免伤害；反应时间极短、小于 1 秒，或驾驶员不在控制回路中，或动作反直觉、空间不足。

C 判断必须说明：感知来源、反应时间、可用操作、道路空间、驾驶员是否在车上、其他交通参与者是否能避让。

## 功能族经验的使用方式

可以使用常识性功能族知识辅助判断，例如照明、制动、转向、驻车、动力等功能的物理作用、驾驶员可感知方式和风险对象。但功能族知识只能回答“这种故障在当前场景中通过什么机制影响风险”，不能直接替代场景推理。

使用功能族经验时必须完成三步：

1. 先从本行场景提取事实：操作状态、环境、速度、风险对象、附加条件、故障如何触发危险事件。
2. 再解释该功能的失效机制如何在这些事实下改变 S/E/C。
3. 最后检查解释中是否引入了本行没有的条件；若有，删除或改写。

不要写成固定规则，例如“某功能某故障通常为 C1/QM”。即便同一故障，在主动请求、未及时发现、驾驶员离车、高速近距离、低速起步、日间低风险等不同场景中，也可能得到不同的 E/C/S。

## ASIL 矩阵

本 skill 使用项目约定的后缀求和规则：

```text
score = S后缀 + E后缀 + C后缀
score <= 6 => QM
score == 7 => A
score == 8 => B
score == 9 => C
score >= 10 => D
```

示例：

- S3 + E4 + C3 = 10 => D
- S3 + E3 + C3 = 9 => C
- S2 + E4 + C2 = 8 => B
- S1 + E2 + C3 = 6 => QM

最终结果必须由脚本校验。
