# SEC和ASIL相关知识
## 严重性分类（S）

### S0：无伤害

定义：无人员伤害。若后果只涉及舒适性、可用性、财产损失或信息体验，不构成人身伤害风险，可评为 S0。

### S1：轻度和中度伤害

定义：轻度和中度伤害。

判断锚点：通常对应低速碰撞或低能量事件造成的轻伤、中等伤害，受伤人员通常可完全恢复或不会出现长期严重后果。

伤害示例：

- 轻度骨折
- 轻微脑震荡
- 擦伤和挫伤
- 软组织损伤

### S2：严重和危及生命的伤害

定义：严重和危及生命的伤害，通常仍有生存可能。

判断锚点：通常对应中等或较高能量碰撞导致的严重伤害，可能需要紧急医疗处理，并可能留下长期后果。

伤害示例：

- 严重骨折
- 内脏损伤
- 严重头部创伤（但可存活）
- 多根肋骨骨折
- 脊柱损伤（非致命）

### S3：危及生命和致命的伤害

定义：危及生命且生存不确定，或致命伤害。

判断锚点：通常对应高速严重碰撞、翻滚、火灾、电击、车辆失控撞击弱势交通参与者等可能导致死亡或不可逆严重伤害的后果。

伤害示例：

- 严重头部创伤（可能致命）
- 多处严重内伤
- 严重烧伤
- 致命碰撞冲击
- 车辆乘员被甩出

## S 等级速度参考表

以下速度区间用于辅助判断车与车、车与道路安全弱势群体碰撞时的严重性等级。使用时应结合相对速度、碰撞角度、车辆质量、是否有保护结构、风险人员位置和逃避空间；不要机械套表。

### 车与车发生碰撞严重性分类（S）的参考速度区间

| Collision Type | S0 | S1 | S2 | S3 |
| --- | --- | --- | --- | --- |
| Front | 0 <= speed < 4 km/h | 4 <= speed < 20 km/h | 20 <= speed < 40 km/h | 40 km/h <= speed |
| Back | 0 <= speed < 4 km/h | 4 <= speed < 20 km/h | 20 <= speed < 40 km/h | 40 km/h <= speed |
| Side | 0 <= speed < 2 km/h | 2 <= speed < 8 km/h | 8 <= speed < 16 km/h | 16 km/h <= speed |

### 车与道路安全弱势群体发生碰撞严重性分类（S）的参考速度区间

道路安全弱势群体是指在交通参与者中自身安全保障性居于弱势的群体，主要包括行人、自行车、电动自行车或摩托车使用者，以及儿童、老人和残疾人。

| Collision Type | S0 | S1 | S2 | S3 |
| --- | --- | --- | --- | --- |
| 与道路安全弱势群体碰撞 | 0 = speed | 0 < speed < 8 km/h | 8 <= speed < 15 km/h | 15 km/h <= speed |


## 暴露度分类（E）

### E0：不可置信

定义：运行场景或运行条件在合理预期使用中不可置信，通常不继续用于 ASIL 分配。使用 E0 必须说明为什么该场景不属于合理预期使用。

示例：

- 仅制造测试模式
- 维修/诊断模式（经销商访问）
- 特定条件下的代客泊车模式
- 很少拖曳车辆的拖车模式
- 温带地区极端天气模式（沙漠/北极）

### E1：极低概率

定义：非常低的发生概率。场景极少出现，通常小于 0.4 小时/年，或对于绝大多数驾驶员小于一年发生一次。

示例：

- 很少使用的拖车或被拖车场景
- 特殊越野驾驶（对于公路车辆）
- 少见的长大陡坡山区驾驶
- 少见的洗车模式或运输模式
- 特定地区才会遇到的极端天气或特殊道路

### E2：低概率

定义：低发生概率。场景较少但可合理遇到，通常 0.4 小时/年 <= x < 4 小时/年，或对于绝大多数驾驶员一年发生几次。

示例：

- 平行泊车操作
- 坡道起步场景
- 少量雨/雪条件驾驶
- 学校区域驾驶
- 偶发停车场复杂会车或狭窄通道

### E3：中等概率

定义：中等发生概率。场景周期性出现，通常 4 小时/年 <= x <= 40 小时/年，或对于一般驾驶员一个月发生一次或多次。

示例：

- 倒车档操作
- 停车场导航
- 住宅区驾驶
- 停停走走的交通
- 乡村道路驾驶
- 郊区驾驶
- 双车道高速公路
- 弯道
- 夜间驾驶
- 混合交通条件

### E4：高概率

定义：高发生概率。场景高频出现，通常大于 40 小时/年，或平均几乎发生在每次驾驶中。

示例：

- 高速公路巡航
- 直线道路驾驶
- 高速驾驶（> 100 km/h）
- 白天驾驶
- 正常天气条件
- 车辆多名乘员
- 对以城市通勤为主的用户，城市/市区驾驶可按高频场景考虑

## 暴露率 E 的定量参考

影响暴露度 E 的参数主要有两个：场景持续时间 d 和场景发生频率 f。

场景持续时间 d 是指车辆运行在被分析场景内的时间占车辆总运行时间的比重。可用小时/年近似表达：

| 指标 | E1 | E2 | E3 | E4 |
| --- | --- | --- | --- | --- |
| 时间范围 d | 极低占比 | < 1% 平均运行时间 | 1%-10% 平均运行时间 | > 10% 平均运行时间 |
| 运行时间（小时/年） | < 0.4 小时/年 | 0.4 小时/年 <= x < 4 小时/年 | 4 小时/年 <= x <= 40 小时/年 | > 40 小时/年 |

场景发生频率 f 是指在一年内车辆运行在所考虑场景的次数：

| 指标 | E1 | E2 | E3 | E4 |
| --- | --- | --- | --- | --- |
| 频率范围 f | 对于绝大多数驾驶员小于一年发生一次 | 对于绝大多数驾驶员一年发生几次 | 对于一般驾驶员一个月发生一次或多次 | 平均几乎发生在每次驾驶中 |
| 基础行驶循环（数量/年） | < 1 次/年 | 1 次/年 <= x < 10 次/年 | 10 次/年 <= x <= 100 次/年 | > 100 次/年 |

使用 d 和 f 判断 E 时，应结合功能适用工况。若持续时间和发生频率指向不同等级，优先选择更能代表该危害场景真实暴露的等级，并在 `E-解释` 中说明理由；不要为了降低或提高 ASIL 任意选择。



## 可控性分类（C）

### C0：一般可控

定义：通常可由相关人员通过常规行为避免伤害，或危害事件不会导致不可接受的人身伤害风险。C0 通常不用于 ASIL 分配。

特征：

- 对驾驶员或相关人员非常明显
- 有充足反应时间
- 简单的纠正动作
- 多种感官提示
- 所需技能极小

示例：

- 单个雨刷停止工作（另一个仍工作）
- 大灯亮度降低（非完全失效）
- 燃油表不准（范围警告仍有效）
- 座椅加热器温度略微偏离目标
- 收音机音量突然变化

所需操作：简单补偿，明显检测

### C1：简单可控

定义：简单可控。大多数驾驶员或相关人员能够通过直接、常规的反应避免伤害。

特征：

- 容易注意到
- 有足够时间反应（> 5 秒）
- 直接的纠正动作
- 清晰的感官反馈
- 普通驾驶员技能即可

示例：

- ABS 降级（单个车轮）
- 动力转向助力减少（非完全丧失）
- 巡航控制需要多次尝试取消
- 自动变速箱延迟换挡（< 2 秒）
- 胎压监测误报警
- 电子驻车制动释放延迟

所需操作：简单反应，普通技能水平

### C2：通常可控

定义：通常可控。部分驾驶员或相关人员可以通过及时、适当的反应避免伤害，但需要更高注意力、空间或操作能力。

特征：

- 注意时可注意到
- 反应时间有限（1-5 秒）
- 需要熟练的驾驶员响应
- 可能需要训练有素的反应
- 需要熟悉的驾驶操作

示例：

- 动力转向完全丧失（行驶中）
- ABS 完全失效（仅能手动制动）
- ESC 降级性能（减少干预）
- 牵引力控制间歇性工作
- 发动机功率降低（非完全丧失）
- 自动变速箱卡在某一档
- 再生制动意外增加

所需操作：熟练驾驶，立即注意

### C3：难以控制或不可控

定义：难以控制或不可控。多数驾驶员或相关人员难以及时采取有效行动避免伤害。

特征：

- 可能不会立即注意到
- 反应时间极短（< 1 秒）
- 需要专业驾驶员技能
- 需要反直觉的响应
- 身体限制（力量/反应）
- 多个同时失效

示例：

- 制动完全失效（所有系统）
- 高速公路速度下转向完全锁死
- 高速公路速度下非预期全速加速
- 弯道中 ESC 非预期强力制动干预
- 行驶中安全气囊意外展开
- 四驱非预期扭矩分配导致打滑
- 高速公路速度下自动转向错误方向
- 悬架突然完全坍塌

所需操作：专业技能，可能不可控

## 判断锚点

判断时必须考虑感知来源、反应时间、可用操作、空间约束、驾驶员是否在车上以及道路使用者是否能避让。告警只能作为因素之一，不能自动等价为可控。

- C0/C1：风险容易被感知，反应窗口充足，普通驾驶员可用直接操作纠正。
- C2：需要及时识别并做出熟练反应，通常仍存在可行控制动作。
- C3：反应时间极短、空间受限、动作反直觉或驾驶员根本不在控制回路中。




## ASIL 等级映射

本项目的最终 ASIL 校验使用 S/E/C 后缀求和规则。

```text
score = S后缀 + E后缀 + C后缀
score <= 6 => QM
score == 7 => A
score == 8 => B
score == 9 => C
score >= 10 => D
```

示例：

```text
S3 + E4 + C3 = 10 => D
S3 + E3 + C3 = 9 => C
S2 + E4 + C2 = 8 => B
S1 + E2 + C3 = 6 => QM
```
