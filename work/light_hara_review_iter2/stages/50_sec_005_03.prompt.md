# S/E/C 和 ASIL 指南

## Severity S

S0：无人员伤害。仅舒适性、可用性、财产损失或信息体验问题。

S1：轻度和中度伤害。通常为低速、低能量事件，可能导致擦伤、挫伤、轻微脑震荡、轻度骨折等，一般可恢复。

S2：严重和危及生命的伤害，但通常仍有生存可能。可能包含严重骨折、内脏损伤、严重头部创伤、多根肋骨骨折、脊柱损伤等。

S3：危及生命和致命伤害。高速严重碰撞、翻滚、火灾、电击、车辆失控撞击弱势交通参与者等可评为 S3。

速度参考只作为锚点，不要机械套用：

- 车车正碰/追尾：小于 4 km/h 常为 S0；4-20 km/h 常为 S1；20-40 km/h 常为 S2；大于等于 40 km/h 可为 S3。
- 车车侧碰：小于 2 km/h 常为 S0；2-8 km/h 常为 S1；8-16 km/h 常为 S2；大于等于 16 km/h 可为 S3。
- 与行人、自行车、电动车、摩托车等弱势交通参与者碰撞：0 为 S0；0-8 km/h 常为 S1；8-15 km/h 常为 S2；大于等于 15 km/h 可为 S3。

## Exposure E

E0：不可置信。场景不属于合理预期使用。

E1：极低概率。通常小于 0.4 小时/年，或绝大多数驾驶员一年不到一次。

E2：低概率。0.4-4 小时/年，或一年几次。例如平行泊车、坡道起步、少量雨雪、学校区域。

E3：中等概率。4-40 小时/年，或月度出现。例如倒车、停车场、住宅区、乡村道路、郊区、弯道、夜间驾驶、混合交通。

E4：高概率。大于 40 小时/年，或几乎每次驾驶。例如直线道路、高速/城市通勤、正常天气、白天、高速公路巡航。

E 取决于操作场景暴露，不取决于故障概率。

## Controllability C

C0：一般可控。伤害可通过常规行为避免，或不会导致不可接受人身伤害风险。

C1：简单可控。大多数驾驶员通过直接常规反应可以避免伤害；提示明显，反应时间通常大于 5 秒。

C2：通常可控。部分驾驶员可通过及时适当反应避免伤害；反应时间通常 1-5 秒，需要较高注意力、空间或操作能力。

C3：难以控制或不可控。多数驾驶员或相关人员难以及时避免伤害；反应时间极短、小于 1 秒，或驾驶员不在控制回路中，或动作反直觉、空间不足。

C 判断必须说明：感知来源、反应时间、可用操作、道路空间、驾驶员是否在车上、其他交通参与者是否能避让。

### 近光灯 HARA 评级提醒

- `近光灯无法点亮`、`亮度过小`、`卡滞在熄灭状态`：夜间刚起步、傍晚行驶或进入暗环境前，驾驶员通常会主动开灯并能发现异常；若场景给不出突然性、不可替代照明和足够接近的风险对象，C 应偏低、结果常为 QM。
- `近光灯点亮响应过晚`：几秒级延迟通常不支持高 ASIL；只有在车辆已经处于低照度危险区域、驾驶员短时间内无法感知或避让风险对象时，才可能提高 S/E/C。
- `近光灯非预期点亮`、`亮度过大`、`卡滞在点亮状态`：近光灯通常弱于远光灯，不能默认造成强眩目。若风险对象能正常识别车辆位置和道路环境，C 应偏低。
- `近光灯请求点亮却熄灭` 是方向错误，不等同于行驶中突然非预期熄灭；驾驶员通常更容易发现请求未达成并采取补救。
- `近光灯请求熄灭却点亮` 通常无人员伤害风险；若进入 HARA，多数场景应为 S0/C0/QM。

## ASIL 矩阵

本 skill 使用项目约定的后缀求和规则：

```text
score = S后缀 + E后缀 + C后缀
score <= 6 => QM
score == 7 => A
score == 8 => B
score == 9 => C
score >= 10 => D
```

示例：

- S3 + E4 + C3 = 10 => D
- S3 + E3 + C3 = 9 => C
- S2 + E4 + C2 = 8 => B
- S1 + E2 + C3 = 6 => QM
